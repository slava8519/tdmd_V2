# `tests/io/fixtures/` — LAMMPS data file fixtures

Fixtures are hermetic: they parse without running LAMMPS so CI stays fast and
offline. The 500-atom LAMMPS-generated fixture called out by exec pack T1.3 is
deferred to T1.11 where the LAMMPS diff harness is already wired; at that point
we will add `al_fcc_500.data` generated by `tools/lammps_smoke_test.in`.

## Files

- `al_fcc_small.data` — 32 Al atoms (FCC, 2×2×2 unit cells, a=4.05 Å), atom_style
  atomic, 1 atom type, box 0–8.1 Å per axis, includes a `Velocities` section.
  This is the happy-path round-trip fixture.
- `corrupt_missing_atoms.data` — declares 4 atoms but omits the `Atoms` section.
  Expected failure: `LammpsDataParseError` complaining about missing `Atoms`.
- `empty.data` — zero bytes. Expected failure: parse error at line 0 ("file is
  empty").
- `triclinic_rejected.data` — orthogonal box plus a `xy xz yz` tilt line.
  Expected failure: parse error "triclinic tilt factors … not supported in M1".

Add a new fixture only when it exercises a distinct failure mode or a new
supported format extension; parity with real LAMMPS output should be added
through integration tests (T1.11) rather than duplicated fixtures.
